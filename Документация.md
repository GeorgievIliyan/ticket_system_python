# Проект 2-ри срок: Билетна система

## Описание

Това е Python приложение за управление на билетна система. Потребителите могат да се регистрират и да влизат, да създават зали, да управляват билети, да генерират QR кодове, да виждат история на продажбите и да управляват виртуална опашка за връщания.

## Използвани технологии

- Python 3
- Peewee ORM
- SQLite база данни
- `qrcode` библиотека
- os и platform за системни операции
- `structures/` за собствена реализация на стек и опашка без външни зависимости

## Инсталация

### Linux / macOS (Anaconda)

1. Създаване на Anaconda среда:

   ```bash
   conda create --name ticket_env python=3.11
   ```

2. Активиране на средата:

   ```bash
   conda activate ticket_env
   ```

3. Инсталиране на зависимостите:

   ```bash
   pip install peewee qrcode pillow
   ```

4. Стартиране на приложението:

   ```bash
   python main.py
   ```

### Windows (Anaconda / PowerShell)

1. Създаване на Anaconda среда:


## Структура на проекта

- `main.py` - главен скрипт, който управлява менюто и навигацията.
- `models/models.py` - дефинира моделите `Account`, `EventHall` и `Ticket` с Peewee.
- `modules/auth.py` - логика за регистрация, вход и изход.
- `modules/halls.py` - CRUD операции за зали.
- `modules/tickets.py` - CRUD операции за билети, търсене, сортиране и генериране на QR кодове.
- `structures/stack.py` - собствен стек за история на последните продажби.
- `structures/queue.py` - собствена опашка за виртуални заявки за връщане.
- `qr_codes/` - папка за записване на генерираните QR изображения.
- `utils/` - помощни модули за валидации и почистване на входни данни.

## Модели

### Account

- `username` - уникално потребителско име.
- `password` - хеширана парола.
- `created_on` - дата на създаване.
- `token` - временен идентификатор след влизане.

### EventHall

- `capacity` - капацитет на залата.
- `rows` - брой редове.
- `created_on` - дата на създаване.

### Ticket

- `price` - цена на билета.
- `date` - дата на събитието.
- `person` - име на посетителя.
- `seat` - място (по избор).
- `row` - ред (по избор).
- `hall` - свързана зала (чужд ключ).

## Основни функции

### Регистрация и вход

- `register()` - създава нов акаунт след въвеждане на потребителско име и парола.
- `login()` - проверява въведените данни и записва токен за сесията.
- `logout()` - премахва токена при изход.

### Управление на зали

- `list_halls()` - показва всички съществуващи зали.
- `create_hall()` - създава нова зала с капацитет и брой редове.
- `update_hall()` - променя параметрите на залата.
- `delete_hall()` - изтрива зала и свързаните билети.

### Управление на билети

- `list_tickets()` - показва всички билети, сортирани по дата.
- `search_tickets()` - търси билет по име на лицето.
- `create_ticket()` - добавя нов билет, избира зала, генерира QR код и записва продажбата в историята.
- `update_ticket()` - редактира данни за билет и обновява QR кода.
- `delete_ticket()` - изтрива билет и свързания QR код.

### Допълнителни функции

- `list_halls_and_tickets()` - комбиниран списък с всички зали и закупени билети.
- `show_recent_sales_history()` - показва история на последните продажби чрез стек.
- `refund_queue_menu()` - управлява виртуална опашка за заявки за връщане.

## Алгоритми и структура на данните

### Стек (Stack)

Стекът е LIFO структура. Последната продажба, която е била регистрирана, се записва най-отгоре на стека. В проекта стекът се използва за история на продажбите:

- `push(item)` добавя нов продаден билет.
- `pop()` връща последната продажба.
- `peek()` вижда последната продажба без да я премахва.

Реализацията е в `structures/stack.py` без външни зависимости.

### Опашка (Queue)

Опашката е FIFO структура. Първият клиент, който е поискал връщане, се обработва първи. В проекта опашката се използва за виртуална линия на клиенти, чакащи възстановяване:

- `enqueue(item)` добавя нова заявка.
- `dequeue()` премахва и връща следващия клиент.
- `peek()` показва следващия клиент без да го премахва.

Реализацията е в `structures/queue.py` без външни зависимости.

### Сортиране по дата

При извеждане на билетите се използва Peewee `order_by(Ticket.date)` за показване на билетите в хронологичен ред.

### Търсене по име

За търсене на билет по име се използва SQL заявка с `fn.LOWER(...).contains(...)`, за да се намерят записи, чието име съвпада частично и без значение от големина на буквите.

## QR кодове

Всеки билет се записва като PNG изображение в папката `qr_codes/`. Генерираният файл съдържа:

- ID на билета
- Име на лицето
- Дата на събитието
- Цена
- Ред и място
- ID на залата
